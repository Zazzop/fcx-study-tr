NSE 7 Olarak Zaten Bildikleriniz (Zaman Harcamayın)

  • Temel Firewall Policy ve NAT kuralları.
  • Basit OSPF komşulukları ve statik routing.
  • SD-WAN temel kuralları (Volume/Session tabanlı yük dengeleme).
  • Security Profile'ların (AV, Web Filter, IPS) standart kullanımı.

FCX İçin Odaklanmanız Gereken "Gap" (Eksik) Alanlar

  • ADVPN + BGP Route Reflector: SD-WAN SLA'lerinin BGP rotalarıyla dinamik olarak nasıl konuştuğu. (Overlay routing).
  • FMG Provisioning: CLI scriptlerini meta-variable'lar ile FortiManager üzerinden yüzlerce cihaza hatasız basmak.
  • ZTNA & EMS Entegrasyonu: SAML IDP kimlik doğrulaması ile Access Proxy üzerinden HTTPS/TCP trafiğini yönetmek.
  • Kasıtlı Hatalar: Sınavda size MTU mismatch olan, asimetrik route edilmiş tüneller verilecek. GUI'den anlamanız imkansızdır, debug flow şarttır.

FCX Survival CLI Commands

Notepad'e Kopyala

1. Asimetrik Routing & Drop Tespiti (Life of a Packet)

# Her zaman filtre ile başlayın, yoksa CPU tavan yapar.
diag debug flow filter addr x.x.x.x
diag debug flow filter protocol 1 # ICMP
diag debug flow show function-name enable # Kernel fonksiyonunu gösterir (Çok kritik!)
diag debug flow trace start 100
diag debug enable
# İşi bitince:
diag debug disable & diag debug flow trace stop & diag debug flow filter clear

2. IPsec VPN Neden Kalkmıyor?

# GUI 'Negotiation Timeout' der ama gerçeği CLI söyler (Örn: Peer ID mismatch, Proposal mismatch).
diag vpn ike log-filter clear
diag vpn ike log-filter dst-addr4 x.x.x.x
diag debug app ike -1
diag debug enable
# Tünel faz 2 durumunu kernel seviyesinde görme:
diag vpn tunnel list name TUNNEL_NAME

3. BGP Komşuluk ve Route İncelemesi

# Komşuluk var ama route neden RIB'e inmedi?
get router info bgp summary
get router info bgp neighbors x.x.x.x advertised-routes
get router info bgp neighbors x.x.x.x received-routes
# BGP paket debug (Update'leri izlemek için):
diag ip router bgp all enable
diag ip router bgp level info
diag debug enable
